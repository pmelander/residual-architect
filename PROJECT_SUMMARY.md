## 🚀 Phase 1 Skills

### 1. Architecture Decision Records (`/adr`)
Create and manage Architecture Decision Records following industry standards.

**Key Features:**
- Standard ADR format
- Automatic numbering
- Status tracking (Proposed → Accepted → Deprecated)
- Search and listing
- Alternatives documentation

**Use Cases:**
- Document technology selections
- Record architecture patterns
- Track decision evolution
- Share context with team

---

### 2. Solution Documentation Generator (`/solution-doc`)
Generate comprehensive technical documentation automatically.

**Key Features:**
- High-Level Design (HLD)
- Low-Level Design (LLD)
- Deployment Guides
- Operations Runbooks
- Complete documentation sets

**Use Cases:**
- New project documentation
- Architecture handoff
- Compliance documentation
- Operations enablement

---

### 3. Technology Stack Advisor (`/tech-stack`)
Recommend and evaluate technology choices with trade-off analysis.

**Key Features:**
- Complete stack recommendations
- Technology comparisons
- Migration analysis
- Risk assessment
- Detailed justifications

**Use Cases:**
- Technology selection
- Migration planning
- Stack evaluation
- Decision support

---

### 4. Design Review (`/design-review`)
Comprehensive architecture reviews across multiple dimensions.

**Key Features:**
- Multi-dimensional scoring
- Critical issue identification
- Best practice validation
- Anti-pattern detection
- Prioritized recommendations

**Use Cases:**
- Pre-production reviews
- Architecture validation
- Security audits
- Performance analysis

## 🔄 Development Workflow

```
User Request → Technology Selection → ADR Creation →
  → Architecture Design → Design Review → Documentation →
    → Implementation → Operations Handoff
```

Each step supported by specific skills:

1. **Technology Selection:** `/tech-stack recommend`
2. **Decision Recording:** `/adr create`
3. **Architecture Design:** `/solution-doc hld`
4. **Design Validation:** `/design-review complete`
5. **Detailed Design:** `/solution-doc lld`
6. **Deployment Planning:** `/solution-doc deployment`
7. **Operations Prep:** `/solution-doc runbook`

## 💡 Use Cases

### Scenario 1: New Project Kickoff
**Time Saved:** 4-6 hours

1. Get stack recommendations: `/tech-stack recommend`
2. Document decisions: `/adr create` (multiple)
3. Create HLD: `/solution-doc hld`
4. Review design: `/design-review architecture`

**Result:** Complete architecture documentation in under 2 hours.

---

### Scenario 2: Pre-Production Review
**Time Saved:** 2-3 hours

1. Comprehensive review: `/design-review complete`
2. Address critical issues
3. Generate runbook: `/solution-doc runbook`
4. Create deployment guide: `/solution-doc deployment`

**Result:** Production-ready with documented operations.

---

### Scenario 3: Technology Migration
**Time Saved:** 8-10 hours

1. Compare options: `/tech-stack compare`
2. Analyze migration: `/tech-stack migrate`
3. Document decision: `/adr create`
4. Plan deployment: `/solution-doc deployment`
5. Review security: `/design-review security`

**Result:** Well-planned migration with risk mitigation.

---

### Scenario 4: Ongoing Architecture
**Time Saved:** 1-2 hours/month

1. Monthly review: `/design-review complete`
2. Update docs: `/solution-doc update`
3. Track decisions: `/adr list` and update
4. Evaluate tech: `/tech-stack evaluate`

**Result:** Architecture stays current and documented.
